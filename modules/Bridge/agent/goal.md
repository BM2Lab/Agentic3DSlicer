# SlicerAgent — Unix Socket Bridge

## Goal
Build an agent framework that controls 3D Slicer via a Unix domain socket IPC bridge.
Treat Slicer manipulation as typed API calls. LLM agents pick actions from a registry;
actions exec code inside Slicer via the socket and return structured results.

---

## Architecture
Four layers (mirrors browser-use):

1. **SlicerAgent** — task loop, LLM turns, step history
2. **SlicerController** — hierarchical namespace registry, auto-generates LLM tool schemas
3. **SlicerSession** — manages Slicer subprocess + Unix socket RPC
4. **SceneStateService** — queries MRML scene, returns compact JSON state each step

## IPC Mechanism

Slicer is launched via `start_slicer.py` (GUI) or `session.py` (headless):
```bash
SLICER_SOCK=/tmp/slicer_agent.sock \
  Slicer --python-script slicer_use/slicer/bootstrap.py --no-splash [--no-main-window]
```

`bootstrap.py` runs a Unix socket server in a daemon thread + a Qt keepalive timer (10ms).
External processes send newline-delimited JSON RPC; bootstrap execs on the Qt main thread
and returns structured results. VTK warnings are captured and forwarded back in the response.

**RPC Protocol:**
- Request:  `{"id": "<uuid>", "code": "<python>"}\n`
- Response: `{"id": "<uuid>", "result": <any>, "stdout": "<str>", "error": "<tb|null>"}\n`

`stdout` includes both Python print output AND VTK warnings (prefixed with `[VTK]`).

## Controller — Namespace Registry

`SlicerController` uses a hierarchical namespace model:

```python
controller = SlicerController()       # module-level singleton in service.py

ns = controller.namespace("volume", "Volume loading, resampling, conversion")

@ns.action("Load a volume file into the scene")
async def load_volume(session, path: str) -> dict: ...
```

**Schema export:**
- `controller.schemas(depth=0)` — namespace names only
- `controller.schemas(depth=1)` — names + descriptions
- `controller.schemas(depth=-1)` — full param schemas (for LLM tool use)

**Agent discovery:**
- `python3 inject.py --list` — human-readable registry
- `python3 fetch_registry.py` — dumps `registry.json` (machine-readable, one-time read for new agents)

## Critical Lessons

| Issue | Fix |
|---|---|
| `--no-main-window` Slicer exits after script | Qt keepalive timer in bootstrap |
| `qt.QTimer.singleShot(0, fn)` from background thread | Use `queue.Queue` drained by keepalive timer |
| `print()` from daemon thread crashes PythonQt | Use file-based `_log()` from background threads |
| VTK warnings invisible to agent | `vtkStringOutputWindow` capture in `capturing_exec` → forwarded in `stdout` response field |
| VTK warnings lost color in Python interactor | Re-emit via `old_vtk_win.DisplayWarningText/ErrorText()` after capture |
| `qMRMLSegmentEditorWidget` crashes headless | Rewrite all 5 effects with numpy/scipy morphology |
| `AddEmptySegment(name)` sets ID not display name | Call `segment.SetName(name)` explicitly after |
| `SetSpacingMode` AttributeError on crop node | Use `SetIsotropicResampling(bool)` |
| `GetIJKToRASMatrix()` TypeError | Pre-allocate `vtk.vtkMatrix4x4()` and pass as arg |
| `controller.call(name=...)` kwarg collision | Renamed param to `action_name` |
| `CreateDefaultVolumeRenderingDisplayNode` missing | Use `CreateVolumeRenderingDisplayNode` |
